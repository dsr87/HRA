Using additional account data

phpSecureSite provides a few functions for fetching account data, you can have a closer look at these in the Authentication and account/group data function reference. However, the account data you can retreive through phpSecureSite is very minimal, and in most cases you would at least want to store a persons full name and email address.

The recommended way to store any additional data for accounts is to create a separate table for this data, with an ID or accountfid foreign key to link it against the account. Then create a few functions for getting this additional data whenever you need to.
